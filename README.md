# ai-team-design-sample

AIで定型業務を「組織化」するための、プロンプト一式とテンプレートのサンプルです。
週次の競合ウォッチ（競合のX公式アカウントを巡回 → レポート化 → 経営層向けスライド化）を題材にしています。

> このリポジトリは、設計の考え方を伝えるための**架空題材のサンプル**です。
> 会社名・競合名・配色はすべて架空（自社 = Northwind）です。実運用では自社・競合の実体に置き換えてください。

📝 解説記事：**プロンプトを磨くのをやめて、就業規則を書き始めた ── AIで定型業務を「組織化」する話**（→ 記事リンク）

## これは何か

「良いプロンプトを書く」ことではなく、「記憶を持たない従業員のために、職務記述書と就業規則と業務マニュアルを整える」という発想で組んだ、小さな業務パイプラインの設計サンプルです。詳しい設計判断は記事を参照してください。ここには、その設計を実際に動かすためのファイルを置いています。

## 構成

```text
ai-team-design-sample/
├── README.md
├── prompt/
│   ├── weekly_x_competitor_prompt.md   # 分析の職務記述（アナリスト役）
│   └── competitor_slides_prompt.md     # スライド化の職務記述（資料作成役）
└── slides/
    └── template.pptx                   # 固定デザイン資産（テキストだけ差し替えて使う）
```

役割で読み替えると、`prompt/` が職務記述書、`template.pptx` がデザインの標準、実行時に生成される週次レポートが業務日誌、`latest.md` が「直近の引き継ぎ資料」を指すポインタにあたります。

## 使い方（概要）

前提として、ブラウザ操作には Playwright 系の MCP、スライド生成には `python-pptx` と、レンダリング確認用に LibreOffice を使う想定です。

1. 作業フォルダ（例：`~/Documents/competitor-watch/`）を用意し、`prompt/` と `slides/template.pptx` を配置する
2. 分析を実行する（アナリスト役）
   `prompt/weekly_x_competitor_prompt.md` を読み込ませ、指示どおりに実行する → レポートと `latest.md` が生成される
3. スライドを生成する（資料作成役）
   `prompt/competitor_slides_prompt.md` を読み込ませ、指示どおりに実行する → `latest.md` を入力に `.pptx` が生成される

### 2工程をまとめて回す（マネージャー役）

順序と引き継ぎ条件を定めた、次の指示文で連結実行できます。検収条件（前工程の成果物が更新されていなければ後工程に進まない）が肝です。

```text
以下を順番に実行してください。

1. prompt/weekly_x_competitor_prompt.md を読み込み、指示に従って実行する
   （レポート生成・保存まで）
2. latest.md の作成日が今日の日付に更新されていることを確認してから、
   prompt/competitor_slides_prompt.md を読み込み、指示に従って実行する
   （pptx生成・保存まで）

ステップ1が失敗した場合（latest.md が更新されていない場合）は、
ステップ2に進まず理由を報告して終了してください。
```

## カスタマイズの勘所

- **調査対象**：`prompt/weekly_x_competitor_prompt.md` の調査対象リストを、自社の競合に置き換える
- **配色**：`prompt/competitor_slides_prompt.md` のブランドカラーパレットと、アクセントカラー（サンプルでは便宜上オレンジ）を、自社・競合の実際の色に置き換える
- **テンプレート**：`slides/template.pptx` のデザインは固定資産。気に入った1枚を作ったら、それをテンプレートとして差し替えれば、毎週同じ品質で再現される
