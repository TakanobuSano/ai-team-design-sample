# ai-team-design-sample

AIで定型業務を「組織化」するための、プロンプト一式とテンプレートのサンプルです。
週次の競合ウォッチ（競合のX公式アカウントを巡回 → レポート化 → 経営層向けスライド化）を題材にしています。

> このリポジトリは、設計の考え方を伝えるための**架空題材のサンプル**です。
> 会社名・競合名・配色はすべて架空（自社 = Northwind）です。実運用では自社・競合の実体に置き換えてください。

📝 解説記事：[**プロンプトを磨くのをやめて、就業規則を書き始めた ── AIで定型業務を「組織化」する話**](https://qiita.com/4q_sano/items/00d02e76cb58254f2c4b)

## これは何か

「良いプロンプトを書く」ことではなく、「記憶を持たない従業員のために、職務記述書と就業規則と業務マニュアルを整える」という発想で組んだ、小さな業務パイプラインの設計サンプルです。詳しい設計判断は記事を参照してください。ここには、その設計を実際に動かすためのファイルを置いています。

## 構成

```text
ai-team-design-sample/
├── README.md
├── prompt/
│   ├── weekly_x_competitor_prompt.md   # 分析の職務記述（アナリスト役）
│   └── competitor_slides_prompt.md     # スライド化の職務記述（資料作成役）
└── slides/
    └── template.pptx                   # 固定デザイン資産（テキストだけ差し替えて使う）
```

役割で読み替えると、`prompt/` が職務記述書、`template.pptx` がデザインの標準、実行時に生成される週次レポートが業務日誌、`latest.md` が「直近の引き継ぎ資料」を指すポインタにあたります。

## 使い方（Claude Code 前提）

このサンプルは **Claude Code** で動かすことを想定しています。Claude Code はカレントディレクトリのファイルをそのまま読み書きするので、このフォルダを置いてプロンプトを読み込ませるだけで動きます。

### 必要なもの
- Claude Code
- ブラウザ操作用の MCP（X の巡回に使用。Playwright 系の MCP など）
- `python-pptx`（スライド生成）
- LibreOffice（生成したスライドを画像化してQA確認するのに使用）

### 配置
作業フォルダ（例：`~/Documents/competitor-watch/`）に、このリポジトリの `prompt/` と `slides/template.pptx` を置きます。生成物（レポート・`latest.md`・スライド）も同じフォルダに溜まっていきます。

### 実行（1工程ずつ）
作業フォルダで Claude Code を起動し、順に読み込ませます。

```text
# 1. 分析（アナリスト役）
prompt/weekly_x_competitor_prompt.md を読み込み、指示に従って実行してください。

# 2. スライド生成（資料作成役）
prompt/competitor_slides_prompt.md を読み込み、指示に従って実行してください。
```

### 実行（2工程まとめて：マネージャー役）
順序と検収条件を定めた次の指示文で連結実行できます。**前工程の成果物（latest.md）が更新されていなければ後工程に進まない**、という受け入れ条件が肝です。

```text
以下を順番に実行してください。

1. prompt/weekly_x_competitor_prompt.md を読み込み、指示に従って実行する
   （レポート生成・保存まで）
2. latest.md の作成日が今日の日付に更新されていることを確認してから、
   prompt/competitor_slides_prompt.md を読み込み、指示に従って実行する
   （pptx生成・保存まで）

ステップ1が失敗した場合（latest.md が更新されていない場合）は、
ステップ2に進まず理由を報告して終了してください。
```

この指示文を `~/.claude/commands/` にカスタムコマンドとして置いておくと、毎回の実行が一語のコマンドで済みます。

> 💡 分析でブラウザ操作のコンテキストが大きくなるため、2工程を1セッションで続けるよりは、工程の合間に `/clear` を挟む（または別セッションで実行する）ほうが、スライドの差し替え精度が安定します。スライド生成は `latest.md` とテンプレートだけを入力にする設計なので、分離しても問題ありません。

## カスタマイズの勘所

- **調査対象**：`prompt/weekly_x_competitor_prompt.md` の調査対象リストを、自社の競合に置き換える
- **配色**：`prompt/competitor_slides_prompt.md` のブランドカラーパレットと、アクセントカラー（サンプルでは便宜上オレンジ）を、自社・競合の実際の色に置き換える
- **テンプレート**：`slides/template.pptx` のデザインは固定資産。気に入った1枚を作ったら、それをテンプレートとして差し替えれば、毎週同じ品質で再現される
